    Abstract base class for readers used in pair-classification. We assume pair classification consists of two texts with one or more outcomes for the pair. Optionally, additional text (such as text common to the pair of texts) can be included as a third text. The pair of texts, optional third text, and outcomes(s) is the instance.

    The jcas representing the instance contains two views. Each text in the pair of texts is set to one of the views. The optional additional text is set to the original jcas of the instance. Information about the instance, such as title and instance id, is set to the original jcas.

    Basic Implementation:

    If you have basic pairs of texts with one possible outcome each, you can create your text pair reader in the following manner:

    • Create a class that extends this class and implements TCReaderSingleLabel. 
       
 import de.tudarmstadt.ukp.dkpro.tc.core.io.AbstractPairReader;
 import de.tudarmstadt.ukp.dkpro.tc.io.TCReaderSingleLabel;
 
 public class MyReader extends AbstractPairReader implements TCReaderSingleLabel {
    • Implement the method getCollectionId() in your class, so it returns the generic corpus name. 
       
 protected String getCollectionId(){
        String collectionId = "MyCorpusName";
        return collectionId; 
 }
    • Implement getLanguage(), returning the language of your corpus.
    • Implement getInitialViewDocId(), returning an ID for this instance, such as " 345&nbsp;12.csv". The id may refer to a filename, or other string in which certain characters neec to be escaped or are not easily human-readable.
    • Implement getInitialViewTitle(), returning a human-readable title for this instance, such as "345part12".
    • Implement getBaseUri() if your text pair originates from a single document in one location, to return the absolute location without the filename, such as "/home/schmidt/MyCorpus/folder3/". This will be used for recursive reading of folders and writing the jcas while preserving directory structure.
    • Implement getText(String), conditionally returning either the first or the second text of your text pair, depending on whether String equals "PART_ONE" or "PART_TWO". 
       
 protected String getText(String part) throws IOException{
        if(part.equals("PART_ONE")){
                return "Here is my text 1.";
        }else if(part.equals("PART_TWO")){
                return "Here is my text 2.";
        }
        return null;
        }
    • Implement TCReaderSingleLabel to return the label for your instance. 
       
 @Override
 public String getTextClassificationOutcome(JCas jcas){
        return "positiveInstance";
 }
    • Then implement getNext(JCas) to attach the classification outcome to the jcas. 
       
 @Override
 public void getNext(JCas jcas)
                throws IOException, CollectionException
 {
        super.getNext(jcas);
 
        TextClassificationOutcome outcome = new TextClassificationOutcome(jcas);
        outcome.setOutcome(getTextClassificationOutcome(jcas));
        outcome.addToIndexes();
 }
    • Implement BaseCollectionReader.hasNext() to return whether or not there are more documents to be read from your collection. Also implement BaseCollectionReader.getProgress() to return the counter of the current document and total number of documents in your collection. 
       
 private List myCorpusFilenames;
 private int currentFile; // start at 0
 
 @Override
 public boolean hasNext(){
        if(currentFile < myCorpusFilenames.size() - 1){
                currentFile++;
                return true;
        }
        return false;
 }
 
 @Override
 public Progress[] getProgress(){
        return new Progress[] { new ProgressImpl(currentFile, myCorpusFilenames.size(),
                        Progress.ENTITIES) }; //i.e., we're on number 6 out of 10 total
 }

    If your text pairs have multiple outcomes each, you should implement TCReaderMultiLabel instead of TCReaderSingleLabel.

      • getBaseUri

        protected abstract String getBaseUri()
        If the pair of texts come from one document, set this to the document's original pathname, without the final filename. It is used if the CAS is written to file. The pathname does not need to actually exist.

        The docID, combined with the BaseURI should be unique for each instance.

        Note: If each of your data instances doesn't have its own source file, then we recommend using an empty string here, or a very simple directory name. Otherwise, DKPro Lab may lose some of your data without loudly notifying you.

        To check for lost data, look in "Evaluation[...]" file and add up the experiment's "Correctly Classified Examples" and "Incorrectly Classified Examples".
